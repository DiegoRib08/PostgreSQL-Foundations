'DELETE FROM courses' without a 'WHERE' clause would delete all rows from the table. To prevent this, I'll always write it as a 'SELECT' query first to see that I'm hitting the desired rows. I also like writing my 'DELETE' queries inside a 'BEGIN;...ROLLBACK;' block alongside a 'SELECT' query inside to confirm the result and one outside to make sure the changes are rolled back while I'm still writing the 'DELETE' query.
